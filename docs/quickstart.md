# Quickstart

Five steps. About ten minutes. Lands you at "I can search my own notes and the assistant knows who I am."

## 0. Pre-reqs

- Python 3.10+ (3.11 recommended).
- `git` (for `memex init`'s `git init` step).
- An OpenAI-compatible LLM endpoint, OR an `OPENAI_API_KEY` in the environment. **For mem0's verbatim writes (`memex mem add`) you don't actually need either** — only LLM-backed flows (`mem learn`, `mem add --infer`) hit the model.

## 1. Install

```bash
git clone <repo-url>
cd memex

python3 -m venv .venv
source .venv/bin/activate
pip install -e .

# optional extras
pip install -e ".[dev]"      # pytest + ruff
pip install -e ".[local]"    # sentence-transformers (offline embeddings; brings torch)
```

Confirm:

```bash
memex --version
# memex 0.1.0
```

## 2. Initialize a root

Pick the profile that fits what you have available:

```bash
# Profile A: OpenAI cloud (default; needs OPENAI_API_KEY)
memex init -u <your-handle>

# Profile B: fully offline embeddings + any OpenAI-compatible LLM endpoint
#            (Ollama, vLLM, LM Studio, LiteLLM, ...)
memex init --profile local -u <your-handle>
```

Either way, after init you'll have:

```
~/memex/
   docs/
      inbox/        projects/    people/
      work/         learning/    reference/
   memex.yaml       <-- edit me to change LLM endpoint, model, budgets, ...
   .kbignore
   .cache/          <-- chroma vectors + mem0 qdrant land here
```

The local profile points the LLM at `http://10.242.29.48:11434/v1` with model `qwen3:4b` by default — edit `~/memex/memex.yaml` to use your own endpoint.

## 3. Add a doc and search it

```bash
echo "# Postgres tuning

## work_mem

Bump to 64MB for analytic queries; keep default for OLTP." \
  | memex doc add - --title "Postgres tuning" --tags db,reference

memex doc search "postgres analytic memory" -k 3
```

Expected output:

```
Postgres tuning  (Postgres tuning; Postgres tuning / work_mem)  score=0.317
  /home/<you>/memex/docs/inbox/postgres-tuning.md  - tags: db, reference
  > # Postgres tuning
  > 
  > ## work_mem
  > Bump to 64MB for analytic queries; keep default for OLTP.
```

What just happened:

```
  add  ->  load markdown
        ->  generate ULID id, sha256 hash, frontmatter
        ->  chunk by H2/H3 (single chunk here; tiny doc)
        ->  embed each chunk via the configured embedder
        ->  upsert into ChromaDB collection 'wiki'

  search -> embed the query
         -> ChromaDB top-k by cosine
         -> BM25 sidecar re-scores
         -> hybrid_alpha-weighted blend
         -> filter by --tag / --since (post-hoc)
         -> render
```

## 4. Tell mem0 something about you

```bash
memex mem add "Prefers TypeScript for new services" --category pref
memex mem add "Lives in Shanghai"                   --category profile
memex mem ls
```

By default `mem add` is **verbatim** — the text you provide is stored exactly as-is with the category you supplied. To opt into mem0's LLM-driven fact extraction (split / merge / dedupe), append `--infer`:

```bash
memex mem add "$(cat meeting-notes.md)" --infer
# or
memex mem learn - < meeting-notes.md      # always uses infer
```

`mem learn` is the right tool for unstructured transcripts; `mem add` is the right tool for "I just want to remember exactly this one fact."

## 5. Build a context block (what Cursor hooks call)

```bash
memex ctx "What's our postgres tuning policy?"
```

Sample output:

```
<!-- BEGIN memex-context (auto-generated by memex ctx) -->
## About the user

- (profile) Lives in Shanghai
- (pref) Prefers TypeScript for new services

## Relevant docs

### [Postgres tuning](/home/<you>/memex/docs/inbox/postgres-tuning.md) - Postgres tuning / work_mem  (score 0.45)
> # Postgres tuning
> 
> ## work_mem
> Bump to 64MB for analytic queries; keep default for OLTP.
<!-- END memex-context -->
```

Hand the block to an LLM and you get an answer grounded in *your* notes and aware of *your* preferences. That is the whole pitch.

## What to do next

- Wire it into Cursor (3 channels, mix and match):

  ```bash
  memex cursor install-hooks                                   # automatic context injection
  memex cursor install-rule .                                  # tell Cursor's agent how to use memex
  memex cursor install-agents --scope user                     # /memex-ask /memex-archive /memex-curator
  ```

  See [cursor.md](cursor.md) for which channel does what.

- Run it as a server (so other machines / containers / LLM tools can hit your memex over HTTP):

  ```bash
  memex serve --port 8000
  open http://localhost:8000/docs   # OpenAPI / Swagger UI
  ```

  Then from elsewhere:

  ```bash
  export MEMEX_API_URL=http://<host>:8000
  memex client status
  memex client doc search "postgres" -k 3
  ```

  See [api.md](api.md) and [docker.md](docker.md).

- Stop downloading models at runtime by **baking them into a Docker image**:

  ```bash
  bash scripts/docker-build-test.sh    # builds + runs full E2E
  ```

  See [docker.md](docker.md).

- Or just read [cli.md](cli.md) for the full command surface.
