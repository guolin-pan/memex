# 快速上手

五步。大约十分钟。结束时你已经能"搜自己写的笔记，并且助手知道你是谁"。

## 0. 准备

- Python 3.10+（推荐 3.11）。
- `git`（`memex init` 会跑 `git init`）。
- 一个 OpenAI 兼容的 LLM 端点，或者环境里有 `OPENAI_API_KEY`。**`memex mem add` 默认逐字写入，根本不调 LLM**——只有 `mem learn`、`mem add --infer` 这类才会用到模型。

## 1. 安装

任选一条路径，最终都得到同一个 `memex` CLI。第一条最快，人和 LLM agent 都能用。

### A. 一键脚本（自动识别 `uv` / `pip`）

```bash
git clone <repo-url>
cd memex
bash scripts/install.sh                  # 默认：装到 .venv/，带 [dev] extras
source .venv/bin/activate
```

脚本是幂等的，从不交互。`bash scripts/install.sh --help` 看完整选项：`--uv`、
`--pip`、`--tool`（uv-tool / pipx 风格）、`--system`、`--extras`、`--venv`、
`--python`。

### B. 手动 + uv（最快，推荐 power user）

```bash
git clone <repo-url>
cd memex

uv venv .venv                            # uv 自动挑 Python
source .venv/bin/activate
uv pip install -e ".[dev]"               # core + pytest/ruff；不要测试就去掉 [dev]
uv lock                                  # 可选：生成 lockfile，可复现安装
```

### C. 手动 + pip

```bash
git clone <repo-url>
cd memex

python3 -m venv .venv                    # 需要 python>=3.10 且带 ensurepip
source .venv/bin/activate
pip install --upgrade pip wheel
pip install -e ".[dev]"
```

### D. 作为最终用户 CLI 安装（不想维护 source 仓库）

```bash
uv tool install <已 clone 的仓库路径>     # uv 的 pipx 等价物，隔离环境
# 或
pipx install <已 clone 的仓库路径>
```

### 验证

```bash
memex --version
# memex 0.1.0
```

可选 extras（和任何路径配合用都行）：

- `[dev]` —— pytest + ruff（贡献代码 / 跑测试）
- `[local]` —— `sentence-transformers`（离线嵌入；会拉 torch，~800 MB）

## 2. 初始化数据根

按你手头的资源选 profile：

```bash
# Profile A：OpenAI 云（默认；需要 OPENAI_API_KEY）
memex init -u <你的-handle>

# Profile B：离线嵌入 + 任意 OpenAI 兼容的 LLM 端点
#            （Ollama / vLLM / LM Studio / LiteLLM ...）
memex init --profile local -u <你的-handle>
```

init 完成后你会有：

```
~/memex/
   docs/
      inbox/        projects/    people/
      work/         learning/    reference/
   memex.yaml       <-- 改这里换 LLM 端点、模型、预算等
   .kbignore
   .cache/          <-- chroma 向量 + mem0 qdrant 落在这里
```

local profile 默认指向 `http://10.242.29.48:11434/v1`，模型 `qwen3:4b`——改 `~/memex/memex.yaml` 指向你自己的端点。

## 3. 加一篇文档，搜一下

```bash
echo "# Postgres tuning

## work_mem

Bump to 64MB for analytic queries; keep default for OLTP." \
  | memex doc add - --title "Postgres tuning" --tags db,reference

memex doc search "postgres analytic memory" -k 3
```

预期输出：

```
Postgres tuning  (Postgres tuning; Postgres tuning / work_mem)  score=0.317
  /home/<你>/memex/docs/inbox/postgres-tuning.md  - tags: db, reference
  > # Postgres tuning
  > 
  > ## work_mem
  > Bump to 64MB for analytic queries; keep default for OLTP.
```

幕后发生的：

```
  add  ->  读 markdown
        ->  生成 ULID id、sha256、frontmatter
        ->  按 H2/H3 切块（本例只有一块；文档太小）
        ->  用配置的 embedder 给每块算 embedding
        ->  upsert 到 ChromaDB collection 'wiki'

  search -> 给 query 算 embedding
         -> ChromaDB 按 cosine 取 top-k
         -> BM25 侧路重打分
         -> 按 hybrid_alpha 加权混合
         -> --tag / --since 后置过滤
         -> 渲染
```

## 4. 告诉 mem0 关于"你"的事

```bash
memex mem add "Prefers TypeScript for new services" --category pref
memex mem add "Lives in Shanghai"                   --category profile
memex mem ls
```

默认 `mem add` 是**逐字存储**——你给什么文本就存什么文本，category 也按你指定的。要让 mem0 用 LLM 做事实抽取（拆 / 合 / 去重），加 `--infer`：

```bash
memex mem add "$(cat meeting-notes.md)" --infer
# 或者
memex mem learn - < meeting-notes.md      # learn 永远用 infer
```

`mem learn` 适合非结构化的会议纪要 / 长聊天；`mem add` 适合"我就要把这一句精确地记下来"。

## 5. 构建一个上下文块（Cursor hooks 调的就是它）

```bash
memex ctx "What's our postgres tuning policy?"
```

样例输出：

```
<!-- BEGIN memex-context (auto-generated by memex ctx) -->
## About the user

- (profile) Lives in Shanghai
- (pref) Prefers TypeScript for new services

## Relevant docs

### [Postgres tuning](/home/<你>/memex/docs/inbox/postgres-tuning.md) - Postgres tuning / work_mem  (score 0.45)
> # Postgres tuning
> 
> ## work_mem
> Bump to 64MB for analytic queries; keep default for OLTP.
<!-- END memex-context -->
```

把这个块给任何 LLM，它就能基于*你的*笔记、知道*你的*偏好来回答。这就是整个项目的卖点。

## 接下来可以做什么

- 接入 Cursor（三种通道，可叠加）：

  ```bash
  memex cursor install-hooks                                   # 自动上下文注入
  memex cursor install-rule .                                  # 教 Cursor agent 怎么用 memex
  memex cursor install-agents --scope user                     # /memex-ask /memex-archive /memex-curator
  ```

  每个通道做什么，看 [cursor.zh.md](cursor.zh.md)。

- 当 server 跑（让别的机器 / 容器 / LLM 工具通过 HTTP 访问同一份 memex）：

  ```bash
  memex serve --port 7963
  open http://localhost:7963/docs   # OpenAPI / Swagger UI
  ```

  然后在别处：

  ```bash
  export MEMEX_API_URL=http://<host>:7963
  memex client status
  memex client doc search "postgres" -k 3
  ```

  详见 [api.zh.md](api.zh.md) 和 [docker.zh.md](docker.zh.md)。

- **把模型烤进 Docker 镜像**，运行时零下载：

  ```bash
  bash scripts/docker-build-test.sh    # 构建 + 端到端测试
  ```

  详见 [docker.zh.md](docker.zh.md)。

- 或直接看 [cli.zh.md](cli.zh.md) 获取完整命令表。
